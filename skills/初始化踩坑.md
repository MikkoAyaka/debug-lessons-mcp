# 初始化踩坑

为当前项目完成 MCP 接入初始化。应在每个新项目中首次使用时调用一次。

## 流程

### 步骤 1：检测项目信息

- **项目标识** (project_id)：从当前工作目录名提取。例如工作目录是 `/Users/xxx/Desktop/my-app`，则 project_id 为 `my-app`
- **当前模型** (model_id)：从当前会话上下文中获取正在使用的模型标识。例如 `claude-opus-4-8`、`deepseek-v4-pro` 等。如果无法确定，询问用户

### 步骤 2：创建项目级 MCP 配置

在项目根目录创建 `.claude/mcp.json`（如已存在则合并，只更新 `debug-lessons` 条目）：

```json
{
  "mcpServers": {
    "debug-lessons": {
      "command": "debug-lessons-mcp",
      "args": [],
      "env": {
        "PROJECT_ID": "<检测到的项目标识>",
        "DB_PATH": "~/.claude/debug-lessons-mcp/data/debug-lessons.db"
      }
    }
  }
}
```

如果 `.claude/mcp.json` 已存在且有其他条目，只新增/更新 `debug-lessons`，保留其余条目不变。

### 步骤 3：安装配套 skills

将以下 4 个 skill 文件复制到 `.claude/skills/` 目录下：

| 文件 | 来源 |
|------|------|
| `记录踩坑.md` | 复制自 `~/.claude/skills/记录踩坑.md` |
| `搜索踩坑.md` | 复制自 `~/.claude/skills/搜索踩坑.md` |
| `踩坑统计.md` | 复制自 `~/.claude/skills/踩坑统计.md` |
| `浏览踩坑.md` | 复制自 `~/.claude/skills/浏览踩坑.md` |

如果 `~/.claude/skills/` 中没有这些文件，则从 npm 包的 `skills/` 目录复制。

### 步骤 4：验证并报告

完成后打印摘要：
- ✅ `.claude/mcp.json` 已配置（项目标识: xxx）
- ✅ 配套 skills 已安装到 `.claude/skills/`
- 📊 数据库位置
- 💡 提示：输入 `/记录踩坑` 开始使用

## 注意
- `.claude/` 目录需要加入 `.gitignore`（避免提交本地配置）
- 如果项目已有 `.claude/mcp.json`，只更新 `debug-lessons` 条目，不覆盖其他 MCP 服务器配置
- 项目标识一旦确定，该项目的所有案例将归属此标识
