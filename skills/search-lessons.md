# Search Lessons / 搜索踩坑

在共享案例库中搜索类似问题的解决经验。

## 流程

1. 用户提供的关键词直接搜索；无参数则从当前对话上下文中提取错误信息作为搜索词
2. 调用 `search_cases`（limit 默认 10）
3. 展示结果：[id] 📌 项目 | 模型 | 分类 | 问题摘要 | 教训摘要
4. 用户可选编号调用 `get_case` 查看详情

---

## 环境触发锚点（重要）

**以下场景必须主动调用 `search_cases`，而非等用户下指令：**

| 触发信号 | 搜索词提取方式 |
|----------|---------------|
| 命令行报错（exit code ≠ 0） | 提取错误消息中的关键词 |
| 编译/构建失败 | 提取编译错误类型 + 文件名/模块名 |
| 运行时异常/崩溃 | 提取异常类型 + 关键堆栈信息 |
| Docker/容器问题 | 提取 Docker 相关术语 |
| 数据库错误 | 提取错误码 + 操作类型 |
| 配置文件不生效 | 提取配置项名称 |
| 用户说"这个错见过没" / "之前怎么解决的" | 用户表述中的关键词 |
| 用户描述一个 Bug | Bug 现象中的关键术语 |

**触发原则：宁可多搜一次，不要让用户重复踩坑。**
搜索结果即使不完全匹配，列出相似案例供用户参考也有价值。

---

## 搜索技巧

- 多关键词空格分隔：`Docker BuildKit 超时` 比 `Docker 构建失败` 精准
- 先用 2-3 个核心词搜索，无结果再用单宽泛词
- 📌 标记 = 当前项目案例（+50 加权排在前面）
- 没找到时提示："案例库暂无相关记录，解决问题后建议用 `/record-lesson` 记录"
